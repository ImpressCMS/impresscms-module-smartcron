Introduction

In order for SmartCron to work, there are a few steps you need to make. First, let's understand what the module does.

The module allows you to create different jobs. Each job has a "type", which is a plugin file decsribing to the module what it needs to do when the job is executed. Currently, the module has a limited amount of job types but we are confident the community's developer will find this module usefull and will create more job types !

So, once jobs have been created, you need to setup a cron job on your server. This cron job will execute a script of the module which will then execute the jobs that need to be executed. So you only need to create 1 cron, and the module will t ake care of everything else !

Setting up the cron job

Cron jobs can be created with different methods. Here is the method for CPanel.
  • Log in to your CPanel
  • Click on the Cron Jobs link
  • Click on the Standard button
  • In the Command to run text box, enter the following line :
  • In the Minute(s) select box, select Every Minute
  • In the Hours(s) select box, select Every Hour
  • In the Day(s)(s) select box, select Every Day
  • In the Month(s) select box, select Every Month
  • In the Weekday(s) select box, select Every Weekday
  • Hit he Save Crontab button

Module permissions

Since the cron job will execute a file of the module while not loged in, Anonymous users need to have the permissions to access the module.

  • Go on System Admin > Groups > Anonymous > Edit
  • Under the Module Access rights, check the box SmartCron
  • Click on the Update Group button

Authorised IP

Since anonymous users have permission to access the module, we need to control how the _execute.php script is executed. In order to acheive that, the module controls the IP adress who are accessing this page.

In the module's preferences, the config IPs allowed to execute the cron jobs needs to be set to the IP of the server which will be authorized to execute the cron job. Please set this preference to your server's IP addres.
